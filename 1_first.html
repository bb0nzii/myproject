■ git 꼭 외워야 하는 용어들
 - Repository : 프로젝트와 그 기록을 추적하는 폴더
 - Clone : 컴퓨터에 원격 저장소의 사본을 만듦
 - Stage : Git에 다음에 저장할 변경사항을 알려줌
 - Commit : 스테이징된 변경사항의 스냅샷을 저장
 - Branch : 동시에 여러 버전이나 기능을 작업
 - Merge : 서로 다른 브런치의 변경 사항을 결합
 - Pull : 원격저장소에서 최신 변경사항을 가져옴
 - Push : 변경사항을 원격저장소로 보냄

 ■ git 명령어(git bash)
 - ls : 현재 폴더에 있는 파일을 나열
 - git config --global "code --wait" : vscode를 기본 편집기로 사용
 - git config --list : 깃에 모든 설정을 나열할 때
 - git config --global unset code editor: 설정 해제

■ 초기 설정 시
 - git config --global user.name "~~~"
 - git config --global user.email "~~~@~~~.com"

■ 구성 수준
 - 시스템(모든사용자) : git config --system
 - 글로벌(현재사용자) : git config --global
 - 로컬(현재저장소) : git config --local
 → 우선순위 : 로컬 글로벌 시스템 순서대로
   로컬(현재저장소) git config user.name "Project Name"
   글로벌(현재사용자) git config --global user.name "Global Name"
   시스템(모든사용자) git config --system user.name "System Name"

1) git 폴더 생성 : mkdir myproject
2) git 초기화 : git init
3) 파일 상태 확인 : git status
4) 파일을 준비 : git add ~~ → 스테이지 영역에 파일을 추가
5) git add --all
6) git status
7) git commit -m "Meowww!!!"
